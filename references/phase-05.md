# Phase 5 · 大纲内容与版式规划

> 可插拔模块。独立升级不影响其他Phase。
> 本Phase将Ghost Deck的标题骨架填充为每页的内容要点，并同步确定每页的版式。
> 内容填充和版式选择必须同步进行：内容类型决定版式，版式反过来约束内容密度。

## 执行步骤

### 步骤1：deck_mode（交付模式）确认

读取Phase 1记录的deck_mode推断，向用户二次确认：

```
基于你的描述，我判断这份PPT的交付模式是 [hybrid/presenting/reading]。

影响：
- 每页信息密度控制在 [60-80% / 40-60% / 80-95%]
- 至少10%的页面为震撼类（单数字大屏/引言等）
- 10页滑动窗口中版式系别多样性≥5种（见 layout-library §节奏约束）

确认[X]，还是调整？
```

交付模式说明：
- presenting（演讲为主）：路演、发布会、演讲，以讲者口头展开为主，页面留白多
- hybrid（混合，默认）：客户汇报、顾问报告，现场讲解+会后带走
- reading（阅读为主）：分析报告、白皮书，脱离讲者也能独立理解

### 步骤2：逐页布局规划卡

**强制等待锚点：** 输出全部布局规划卡后，必须停止，等待用户回复确认后才能进入 Phase 6。不得在用户确认前自动继续生成任何设计内容。

为每一页生成布局规划卡（坐标字段来自 `grid-system.md`，Phase 7 直接使用不再计算）：

```
页码：[X]
Action Title：[标题]
叙事目的：[impact/structure/explain/evidence/transition/action]
视觉主角：[大数字/对比关系/流程图/数据图表/矩阵框架/引言文字/结构分解]
内容要点：
  · [要点1]（≤[单行字数上限]字）
  · [要点2]
  · [要点3]
数据来源：[如有，列出需要标注的来源]
推荐版式：[版式ID]
  骨架文件：svg-skeleton-[系].md → [版式ID]章节
  区域坐标：[从grid-system.md复制的精确坐标，如"左栏 x=40,y=100,w=590,h=580"]
备选版式1：[版式ID]
备选版式2：[版式ID]
```

**坐标字段是Phase 7的设计依据，Phase 7不再自行决定坐标。**

**按视觉档（theme）选结构性页模板（来自 Step 1.5，Phase 1 已记录）：**
- A / B 档：封面/目录/章节用 `themes.py` 的 `cover/toc/section`，内容页用 `chrome` 皮肤（B 档含图标/关系图组件）。
- C 高表现力档：封面/章节/金句改用深色重音模板 `cover_dark/section_dark/quote_dark`，内容页仍用 `chrome`(B 皮肤)，数据页用 `charts` 组件占主导，并按 Phase 4 规划穿插"白—白—深"节奏。
- 在每页布局规划卡的"推荐版式"旁注明本页所用 theme 模板/组件，供 Phase 7 直接调用。

版式选择规则（读 `layout-library.md`）：
- 视觉主角=大数字/关键指标/震撼结论 → **I系**（Impact）→ I-01 / I-04 为首选
- 视觉主角=对比/现状vs目标 → **I-03** 或 **S-08**
- 视觉主角=紧迫性/时间压力 → **I-05**
- 视觉主角=并列要点/矩阵/时间轴 → **S系**（Structure）→ S-01~S-12
- 视觉主角=流程/步骤/价值链 → **E-03**（流程示意图）或 **S-03**（时间轴）
- 视觉主角=图文说明/概念图/架构 → **E系**（Explain）→ E-01~E-04
- 视觉主角=数据图表 → **V系**（Evidence）→ V-02 / V-03 / V-04
- 视觉主角=多KPI看板 → **S-09** 或 **V-01**
- 视觉主角=引言/深色底震撼 → **T-01** 或 **I-02**
- 视觉主角=章节切换 → **T-03**
- 视觉主角=层级/树状/金字塔 → **G系**（Graph）→ G-01 / G-02 / G-03
- 视觉主角=决策/行动/责任 → **A系**（Action）→ A-01 / A-02 / A-03
- 封面/分隔/结尾页 → **SPC系** → SPC-01 / SPC-02 / SPC-03

> 完整版式清单及每个版式的骨架文件见 `layout-library.md`。版式ID即骨架文件章节名，如 I-01 对应 svg-skeleton-i.md 的 `## I-01` 章节。**例外**：SPC 系（封面/分隔/结尾）骨架在 `svg-skeleton-common.md` 末尾，Ch 系（数据图）用 `charts.py` 组件——不要去找 svg-skeleton-spc.md（不存在）。

### 步骤2.5：图形化优先 + 数据强制图表化（v2.2 · 防平淡）

每页布局规划卡定稿前逐页执行：

1. **数据强制图表化**：若该页聚集 **≥3 条 D 类数据**（Phase 2 标注、带数值），或含可对比/可排序/时间序列的数字 → **该页必须用数据图表，禁止用纯文字卡或纯表格承载多条数据**。按形态选 charts 组件：对比/排名 → `bar`/`hbar`、趋势 → `line`、增减拆解 → `waterfall`、转化收窄 → `funnel`、分期排期 → `gantt`、实际vs目标 → `bullet`、构成 → `stacked`、前后对比 → `slope`（9 类均已在 charts.py 实现，一行调用）。版式记 V 系并写明调用的组件名。
2. **图形化优先**：非数据页先判断能否用结构图（E 系流程/架构、G 系树/金字塔）或关系图组件承载；能则优先。**纯文字排列版式（S-01/S-04/S-05/S-07/S-10/S-11）退为兜底**，选它须在规划卡写明“此页无可图形化内容”。
3. **图形化占比下限**：连续纯文字排列版式不超过 2 页；每 8 页内 ≥2 页含数据图表（Ch/V 系）或结构图（E/G 系）；全册数据图表页 ≥ ⌈内容页数 / 6⌉。低于线 → 回查是否有 D 类数据被文字化。

---

### 步骤3：信息密度控制

根据deck_mode设置每页内容密度：

| deck_mode | bullets上限 | 每条字数上限 |
|-----------|------------|------------|
| presenting | ≤3条 | ≤15字 |
| hybrid（默认） | ≤5条 | ≤20字 |
| reading | ≤7条 | ≤30字 |

一页一消息强制检验：
每页只传递一个核心信息。标题中含「和/与/并且/同时」→ 标记为需要拆页。

---

### 步骤4：视觉节奏检查

全部规划卡生成后执行（阈值唯一源见 `layout-library.md §节奏约束`）：
- 连续雷同检测：连续3页不能使用相同版式系别
- 震撼类检查：每10页至少1页impact类
- 过渡类检查：每8-10页至少1页transition类
- 多样性统计：10页滑动窗口内，版式系别种类≥5种

## 跳过风险

```
风险等级：中
风险说明：
  跳过版式规划，Phase 7设计时即兴选版式，
  容易导致全PPT视觉单调（只有上下分/左右分）。
  更大的风险：内容和版式不匹配，设计阶段大量返工。
  在这里批量确认版式，比设计后修改快得多。
```
