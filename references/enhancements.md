## 一、转换器 / 工具链修复（P0，影响每一份 deck）

1. **文本宽度估算偏窄 → 文字折行**（已修，见 `assets/svg_to_pptx/drawingml_utils.py` 的 `estimate_text_width`）
   - 原因：旧估算把 `1 i l j ! |` 记 `0.3×fs`、多数拉丁字符 `0.55×`，普遍低估；
     叠加 LibreOffice 对 `wrap="none"` 不严格遵守，框一窄即折行（编号"01"、长英文串等被拆行）。
   - 修法（过宽无害：单行 + 锚定不变，宁可偏宽不可偏窄）：
     窄字符 `0.3→0.5`、基准 `0.55→0.62`、`mMwWOQ` `0.75→0.82`、CJK `1.0→1.02`、bold `×1.06`、整体 `×1.08`。
2. **页码用 `ORDER` 列表驱动**：维护一个页面顺序列表，出片时统一重排页码，不把页码写死在每页 —— 插页/改序不再逐页手改。
3. **产物写"版本目录"**（每轮新目录，如 `out/v1/ v2/ ...`），不原地覆盖 —— 规避产物文件被占用/锁定导致的写入失败。

## 三、设计规则（写进规范 + QC 清单）

1. **强调色 = 信号，不是装饰**：同类并列项要么全中性、要么完全一致；
   主色只用于 当前 / 推荐 / 阶段 / 主操作 / 单个高亮数据 / 品牌点。
   → QC 增一条：**"页面内是否存在无语义的单点强调色？"**（有则改中性）
2. **真实照片用占位框**：用带真实比例的"现场实拍 · 待替换"虚线框（`placeholder`）；**禁 AI 生成图 / 通用网图**。
3. **样张先行**：任何大改视觉语言后，先出 3–4 张代表性样张与用户确认，再铺全册（强化 phase-06 样稿确认）。

## 四、内容方法论（建议纳入 phase-02 / 03 提示语）

- **一手材料优先**：用原始转写 / 原文，不只信二手纪要；关键事实标来源；不确定标"待确认"。
- **措辞分寸**（B2B 客户方案通用）：不让方案显得"砍范围 / 省钱"；不暗示否定客户既有规划；正向表达。
- **Show, don't tell**：价值主张用客观排序 / 结构表达，不写自夸口号。
- **脱敏为出片前固定步骤**：去人名、去第三方公司名（换角色 / 同行 / 某企业）；交付前全局扫一遍敏感词。
- **流程先确认形态**：涉及工艺 / 流程 / 防错时，先确认对象真实形态（人工 vs 自动化等），推断细节标"待调研"。
