# Layout Library · 版式库

> 本文件被 Phase 5（版式规划）和 Phase 7（逐页设计）调用。
> 版式按视觉主角类型分为 A-G 七类，每类包含1个推荐版式和2个备选版式。
> 版式选择依据：这页的核心视觉元素是什么（视觉主角），而不是叙事目的。

---

## 版式选择逻辑

**第一步：确定视觉主角类型**

问自己：这页最想让受众看到/记住的是什么？

| 视觉主角 | 对应版式类别 |
|---------|-----------|
| 一个或几个关键数字/指标 | A类：数字聚焦型 |
| 两个元素的对立/差距/对比 | B类：对比张力型 |
| 时序/步骤/阶段关系 | C类：流程叙事型 |
| 数据图表本身 | D类：数据图表型 |
| 分类/优先级/评估矩阵 | E类：矩阵框架型 |
| 文字本身的力量（引言/标题） | F类：引言震撼型 |
| 层级/组成/分解关系 | G类：结构分解型 |

**第二步：在类别内选推荐或备选**

推荐版式是最常用的，优先选择。
备选版式用于节奏调剂（避免同类版式连续出现）。

---

## A类 · 数字聚焦型

视觉主角 = 单一大数字或关键指标。受众眼神第一秒落在数字上。

### A-01 · 全屏震撼数字（推荐）
```
布局：数字占全屏60%+，一行注释，极简留白
适用：单一核心指标、ROI数字、里程碑数据
特征：数字极大，注释极短，留白极多
```

### A-02 · 数字左+解释右（备选1）
```
布局：左侧60%大数字，右侧40%文字注解（3-4行）
适用：需要对数字做补充说明的场景
特征：数字主导，文字辅助解释
```

### A-03 · 三指标并列（备选2）
```
布局：三个KPI横排，各占1/3，下方共同注释
适用：需要同时展示3个相关指标
特征：三指标平行，无主次之分
```

---

## B类 · 对比张力型

视觉主角 = 两个元素的对立、差距或选择关系。

### B-01 · 现状vs目标（推荐）
```
布局：左右各半，左灰右蓝（或品牌主色），中间虚线
适用：现状与理想状态的对比，改变前后的对比
特征：左侧偏灰暗，右侧高亮，方向感明确
```

### B-02 · 问题→解法（备选1）
```
布局：左红右绿，中间→箭头
适用：问题与解决方案的映射
特征：强语义色，方向感强，适合说服性汇报
```

### B-03 · 方案选项比较（备选2）
```
布局：2-4个选项上下堆叠，推荐选项高亮
适用：多方案评估，需要指出推荐方向
特征：推荐方案视觉突出，其余方案弱化
```

---

## C类 · 流程叙事型

视觉主角 = 时序关系、步骤顺序或阶段推进。

### C-01 · 横向步骤流（推荐）
```
布局：3-5步横排，箭头连接，当前或重要步骤高亮
适用：实施计划、执行步骤、流程说明
特征：左→右方向感，步骤清晰，高亮标注关键步骤
```

### C-02 · 时间轴纵向（备选1）
```
布局：左侧垂直轴，右侧事件描述，当前位置标注
适用：历史发展、项目里程碑、时间线展示
特征：上→下时序，可标注过去/现在/未来
```

### C-03 · 漏斗/箭头流程（备选2）
```
布局：3个阶段箭头形状横排，强调方向和流动感
适用：转化漏斗、价值传递链、因果链条
特征：视觉冲击力强，阶段间的递进关系明确
```

---

## D类 · 数据图表型

视觉主角 = 图表本身，文字为注脚。

### D-01 · 图表全幅主体（推荐）
```
布局：图表占页面70%+，标题+注释贴边
适用：数据量大、趋势明显、需要受众自己读图的场景
特征：图表是绝对主角，文字只做标注
禁用：饼图、3D图表、渐变填充
```

### D-02 · 图表左+洞察右（备选1）
```
布局：图表占60%，右侧关键数字+2-3行解读
适用：图表有明确结论，需要引导受众看到重点
特征：图表+关键数字组合，避免受众自己得出错误结论
```

### D-03 · 主图左+辅图右（备选2）
```
布局：主图左侧，右上次要图+右下数据表格
适用：需要多维度展示同一议题
特征：主次关系明确，密度较高，适合reading模式
```

---

## E类 · 矩阵框架型

视觉主角 = 分类关系、优先级排列或多维评估。

### E-01 · 2×2优先级矩阵（推荐）
```
布局：两轴四象限，圆圈代表选项（圈大=影响大），高价值区高亮
适用：优先级决策、风险/收益分析、战略选择
特征：直观呈现选项的相对位置，避免纯文字描述
```

### E-02 · 多方案评分列（备选1）
```
布局：4列对比，每列一个方案，推荐列高亮，评分维度行
适用：方案比选、供应商评估、工具选型
特征：结构化对比，推荐方案视觉突出
```

### E-03 · 价值/复杂度矩阵（备选2）
```
布局：2×2矩阵，高价值低复杂度象限高亮
适用：功能优先级、项目排期、改进机会评估
特征：高亮"快赢区"，弱化低优先级区域
```

---

## F类 · 引言震撼型

视觉主角 = 文字本身的力量，极简留白。

### F-01 · 结论引言全屏（推荐）
```
布局：大字居中，极简留白，上下各一条细线
适用：章节起首、关键结论页、震撼性判断
特征：一句话，大字，大留白，视觉冲击力强
```

### F-02 · 章节分隔页（备选1）
```
布局：左侧品牌色竖条，章节号+标题+2-3行摘要
适用：多章节PPT的章节切换
特征：视觉节奏调剂，提示进入新主题
```

### F-03 · 引用语录页（备选2）
```
布局：大引号，核心引用文字，出处小字
适用：专家观点、客户反馈、权威数据引用
特征：引用内容突出，来源清晰，增加可信度
```

---

## G类 · 结构分解型

视觉主角 = 层级关系、组成关系或分解结构。

### G-01 · 树状分解图（推荐）
```
布局：顶层节点居中，向下展开3个分支，可再展开一层
适用：MECE框架展示、问题分解、价值链分析
特征：层级清晰，顶层高亮，适合展示整体→部分关系
```

### G-02 · 三支柱并立（备选1）
```
布局：三等分竖列，各支柱含标题+2-3个子要点
适用：三大支撑、三个价值维度、三阶段策略
特征：三者并列平等，无先后顺序，强调整体性
```

### G-03 · 层级金字塔（备选2）
```
布局：梯形层叠，顶层高亮最重要，底层最宽
适用：战略/管理/执行三层、马斯洛式层级、能力模型
特征：层级感强，顶层最突出，底层最基础
```

## 版式选用限制

**节奏规则（Phase 5视觉节奏检查时执行）：**
- 连续3页不能使用同一类别的版式
- 10页滑动窗口内，版式类别种类≥6种
- 每10页至少1页F类（引言震撼型）作为impact页
- 每8-10页至少1页F-02（章节分隔页）或F-01作为transition页

**绝对禁用（所有版式通用）：**
- 渐变填充（任何元素）
- 3D效果（任何图表或形状）
- 投影效果（drop shadow）
- 纯黑色 #000000（用近黑代替）
- 饼图（任何形式）
- 独立图例（颜色释义直接标注在图上）
