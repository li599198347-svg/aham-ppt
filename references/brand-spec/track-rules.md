# Aham UI v6.1 — 轨道分流与统一规则

> **核心结论**：Aham UI v6.1 不再按"作品类型"分轨，而是按**落地介质**分四轨——**网页 / 应用 (macOS) / Office / 邮件**。同一套 token（`tokens.css`）跨四轨共享，差异只在落地介质强制的取值（字号单位、栅格、可用控件）。
>
> **本技能只产出 PPT，归属 `Office 轨 · PPT 子模型`。** 取值见 `tokens.css` 与 `brand.md`，落地细则遵循 `aham-ui-office.md` 的 PPT 段。本文件把"分轨思想 + 跨轨一致铁规 + 团队实操建议"显性化，避免混用介质导致取值串档。

---

## 0. 三条根本原则（跨所有轨道，永不动摇）

无论网页 / 应用 / Office / 邮件，下列三条恒定：

1. **单一色相 = 蓝仅点缀**：全系统唯一彩色为 `--accent #336EE8`，其余皆三层表面灰 + 四级墨色。蓝**绝不铺底**，只用于：logo / 主操作（发送）/ 选中下划线 / 推荐标 / 单个高亮数据点。「蓝若第一眼就被注意到，即超标。」
2. **状态双通道 = 点 + 文字**：状态恒为 **6px 圆点 + 文字标签**成对出现，颜色仅作辅助（成功 `#5A7A60` / 警示 `#8A7333` / 风险 `#9E3D31`，仅真风险）。禁红黄绿灯，禁实色填充药丸，禁只靠颜色区分。
3. **禁用清单（全轨共享，零例外）**：
   渐变 · 投影（静置/hover，仅浮层例外）· 3D / 拟物 / 毛玻璃 · emoji · 饼图 / 环形图 · 衬线字体 · 第二装饰彩色 · 纯黑 `#000000`（最深用 `--ink-1 #262626`）· 表格竖线 · 蓝铺底 · 红黄绿灯。

---

## 1. 四介质轨的边界

四轨按**作品最终落地在哪种介质**划分，**互不重叠**。选错介质轨，字号单位、栅格、可用控件全跟着错。

| 介质轨 | 落地介质 | 单位 / 栅格 | 字体 | 表面 | 落地细则 |
|---|---|---|---|---|---|
| **网页** | 浏览器 Web / 后台 | px · 4 基栅格 | Inter / 雅黑·PingFang；数字 JetBrains Mono | 三层 `#FFFFFF / #F3F3F3 / #E7E7E7` | 见 `aham-ui-web` 段 |
| **应用 (macOS)** | 桌面原生应用 | pt · macOS HIG | 同上（system-ui 回退 -apple-system） | 同三层 | 见 `aham-ui-app` 段 |
| **Office** | PPT / Word / Excel | pt | 雅黑·PingFang；数字 Consolas（mono 回退） | 底永远纯白 | **本技能落点**，见下 §1.1 |
| **邮件** | HTML 邮件 / 纯文本 | px · 邮件安全子集 | 雅黑·PingFang web-safe 回退 | 纯白 | 见 `aham-ui-office.md` 邮件段 |

> **绝不混用介质**：PPT 里塞 px 栅格 = 错；Web 后台用 PPT 的 44pt 大数字 = 错；邮件里引浏览器才支持的 CSS = 错；任意轨出现衬线 / 蓝铺底 / 红黄绿灯 = 错（违反 §0）。

### 决策树

```
作品最终在哪里被看到？
├─ 浏览器里跑的页面 / 后台          → 网页轨
├─ macOS 桌面原生应用              → 应用 (macOS) 轨
├─ 发给客户的 PPT / Word / Excel   → Office 轨   ←（本技能 = PPT 子模型）
└─ 收件箱里的 HTML / 纯文本邮件      → 邮件轨
```

### 1.1 本技能 = Office 轨 · PPT 子模型（权威取值）

PPT 子模型在 §0 三原则之上，强制以下落地取值（源：`tokens.css` §10 + `aham-ui-office.md` PPT 段）：

| 维度 | 取值 |
|---|---|
| 画幅 | **16:9**（1280×720 设计基准） |
| 底色 | **永远纯白 `#FFFFFF`**，禁深底 / 渐变 / 蓝铺底 |
| 正文 / 要点 | **14–16pt**（`--ppt-body 15pt`），左对齐 |
| 内容页标题 | **24pt** `--ppt-title`，`--ink-1` 加粗 |
| 封面标题 | 32–40pt（`--ppt-cover 36pt`），左对齐不居中堆叠 |
| 大数字 | **Consolas 44–60pt**（`--ppt-bignum 54pt`），等宽 |
| 页脚 / 来源 | 9pt `--ppt-footer`，`--ink-3` |
| 蓝点缀 | 仅 logo / 推荐标 / 选中下划线 / **单个**高亮数据点；其余灰阶 |
| 图表 | **灰阶 + 一抹蓝**：柱/线 `--chart-bar #9B9B9B` / `--chart-bar-2 #C8C8C8`，重点项一个 `--chart-hi #336EE8`；轴 `#E7E7E7`，标签 `#6E6E6E`。禁饼图/环形图 |
| 表格 | 只横线，无竖线；表头加粗 `--ink-2`；数字右对齐等宽 Consolas |
| 状态 | 6px 点 + 文字（成功/警示/风险三色），禁药丸/红黄绿灯 |
| 推荐列 | 选中 = 扁平灰 `--surface-2 #F3F3F3` 底，**非蓝底**；推荐标 = 蓝字 + 短蓝下划线 |

## 2. 跨介质一致的「铁规」（100% 强制，不分轨道）

✅ **必须做**
- 任何文字 vs 背景对比度 ≥ 4.5:1（WCAG AA）；正文用 `--ink-1`，次要 `--ink-2`，注释/页脚 `--ink-3`。
- 所有数字 / ID / 金额 / 时间用等宽：屏幕 `JetBrains Mono`，Office `Consolas`（mono 回退）。
- 中英文之间留半角空格；数字与单位之间不留空格（`45 天` / `30%`）。
- 状态用 6px 点 + 文字双通道（§0-2）。
- 深度靠三层表面层差 + 1px 细线（`--surface-3 #E7E7E7`）表达，**留白优先于线，线优先于阴影**。
- 层级靠字号 + 字重，不靠衬线、不靠颜色。

❌ **绝不做**（与 §0-3 禁用清单同源，全轨强制）
- 渐变 / 投影（静置·hover）/ 3D / 拟物 / 毛玻璃 / 光晕。
- emoji（任何作品内）。
- 饼图 / 环形图 / 3D 图表。
- 衬线字体（标题正文一律单一无衬线）。
- 第二装饰彩色（除三语义色 + 唯一蓝点缀）。
- 纯黑 `#000000`（最深 `--ink-1 #262626`）。
- 表格竖线。
- 蓝铺底（蓝只点缀）。
- 红黄绿灯式状态着色。

## 3. 共用 Token 命名约定（按「角色」取名）

CSS 变量按**角色**而非颜色名取名，四轨共享同一套（权威源 `tokens.css`）：

```css
/* 表面 —— 三层，全轨共享 */
--surface-1 #FFFFFF   /* 内容 / 卡片 / 文档底 / PPT 底 */
--surface-2 #F3F3F3   /* 面板 / 选中列底（非蓝）*/
--surface-3 #E7E7E7   /* 线 / 分隔 / 禁用 */

/* 墨色 —— 四级，全轨共享（禁纯黑）*/
--ink-1 #262626  --ink-2 #6E6E6E  --ink-3 #9B9B9B  --ink-4 #C4C4C4

/* 强调 —— 唯一蓝点缀，全轨同一值 */
--accent #336EE8       /* logo / 主操作 / 选中下划线 / 推荐标 / 单高亮数据点 */

/* 语义 —— 仅真风险 + 双通道 */
--success #5A7A60   --warning #8A7333   --danger #9E3D31

/* 字体 —— 单一无衬线 + 等宽，禁衬线 */
--ff-sans   /* Inter → 微软雅黑 / PingFang SC → sans-serif */
--ff-mono   /* JetBrains Mono → Consolas（Office）→ monospace；CJK 禁入 */

/* PPT 字号（Office 轨 · pt）见 tokens.css §10 */
--ppt-cover 36pt  --ppt-title 24pt  --ppt-body 15pt  --ppt-bignum 54pt  --ppt-footer 9pt

/* 图表灰阶 + 一抹蓝 */
--chart-bar #9B9B9B  --chart-bar-2 #C8C8C8  --chart-axis #E7E7E7  --chart-hi #336EE8
```

→ 取值以 `tokens.css` 为单一事实源；任何轨道不允许写死颜色。

## 4. 给团队的实操建议

1. **开工前先认介质轨**：在文件顶部注释落地介质 `<!-- track: Office · PPT 子模型 -->`，下游能立刻判断该用哪套取值。
2. **永远引 token，不写死颜色**：用 `var(--accent)` / `var(--ink-1)`，禁直接敲十六进制。
3. **跨轨复用时替换取值，不串档**：PPT 模板搬到 Web 必须把 pt 换 px、把 Consolas 换 JetBrains Mono；反之亦然。绝不让两套单位 / 两种 mono 同时出现。
4. **AI Prompt 起手即点名轨道**：例如「请按 Office 轨 · PPT 子模型生成：16:9、纯白底、正文 15pt、标题 24pt、大数字 Consolas 54pt、图表灰阶 + 一个 `#336EE8` 高亮项、状态用 6px 点 + 文字」。
5. **自检三问**：① 蓝是不是第一眼就跳出来了？（是则超标）② 有没有衬线 / 纯黑 / 表格竖线 / 红黄绿灯？（有则违规）③ 状态是不是点 + 文字双通道？（否则不合格）
